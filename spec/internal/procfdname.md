## __procfdname (内部函数)

```c
// C 声明 (syscall.h)
hidden void __procfdname(char __buf[static 15+3*sizeof(int)], unsigned fd);
```

```rust
// Rust 对应声明 (rusl)
// 注：缓冲区最小长度 = 15 + 3 * sizeof(int)，对于 32 位 int 为 27 字节
unsafe fn __procfdname(buf: *mut u8, fd: c_uint);
```

**[Visibility]: Internal (不导出)** — musl 内部辅助函数，用于构造 `/proc/self/fd/` 路径字符串。被 `syscall.h` 声明，供 fstat/fchdir 等通过 /proc 实现 fd 操作的模块使用。POSIX/C 标准未定义。

---

### 前置条件

1. `buf` 指向至少 `15 + 3*sizeof(int)` 字节的已分配缓冲区（对于 32 位 `int` 为 27 字节，对于 64 位 `int` 则为 27 字节——注意 `int` 在 LP64 下仍是 32 位）
2. `fd` 为有效的文件描述符编号（非负整数）或 `AT_FDCWD` 特殊值

### 后置条件

**Case 1 — fd == 0**:
- `buf` 内容为 `"/proc/self/fd/0"`（NUL 终止）
- 该路径总长 = 15 字符（前缀 14 + '0' + NUL）

**Case 2 — fd > 0**:
- `buf` 内容为 `"/proc/self/fd/N"`，其中 `N` 是 `fd` 的十进制字符串表示（NUL 终止）
- 无前导零
- 路径长度为 15 + floor(log10(fd)) + 1（不含 NUL）

**Case 3 — 任意 fd 值**:
- `buf` 始终以 NUL 字符终止
- 不修改除 `buf` 外的任何全局或局部状态

### 系统算法（System Algorithm）

该函数使用 **两遍扫描** 策略避免反转或临时缓冲区：

1. **前缀复制**: 从字符串字面量 `"/proc/self/fd/"` 逐字符拷贝到 `buf`，得到前缀 `/proc/self/fd/`，`i` 停在前缀末尾索引处（14）

2. **特判 fd == 0**: 直接写入字符 `'0'` 和 NUL 终止符，直接返回

3. **第一遍 — 测量数字位数**: `for (j=fd; j; j/=10, i++);`，不写入任何内容，仅计算 `fd` 的十进制位数并将 `i` 推进到最终 NUL 位置

4. **NUL 预置**: `buf[i] = 0` 在最终 NUL 所在位置预先写入终止符

5. **第二遍 — 反向填充数字**: `for (; fd; fd/=10) buf[--i] = '0' + fd%10;`，从右向左逐位填充数字字符，自然获得正确的数字顺序

**意图（Intent）**:
该实现在 O(log10(fd)) 时间内完成，不使用除法反转、字符串反转、临时数组或 snprintf。两遍扫描的技巧是：第一次仅计算位数推进索引指针，第二次反向填充数字，利用了 `%10` 取最低位、`--i` 向前移动写入位置的自然配合，避免了逆序输出问题。

### 不变量（Invariants）

1. 循环不变量（第一遍）: `j` 初始等于 `fd`，每轮 `j/=10`，循环次数等于 `fd` 的十进制位数
2. 循环不变量（第二遍）: 在每轮开始前，`buf[i]` 是上一轮设置的 NUL（于第一轮）或已填充的数字字符
3. 路径前缀 `/proc/self/fd/` 占 14 个字节（不含 NUL），`buf[0..13]` 在整个函数执行过程中不被修改
4. `buf` 输出总是有效的 NUL 终止 C 字符串
