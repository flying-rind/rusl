# wide.h 规约

> musl libc 内部 Unicode 字符宽度查找表。定义两级索引表 `wide.h`，用于 `wcwidth` 等函数快速查询字符的显示列宽。

---

## 数据结构

### 查表数组

```c
static const unsigned char wide[] = { ... };
```

[Visibility]: Internal — musl ctype 内部查找表，编译时嵌入 `.rodata`，不对外导出

#### Intent

存储 Unicode 字符的终端显示列宽属性。字符宽度有三个可能值：
- `0` — 零宽字符（组合标记、零宽连接符等）
- `1` — 普通字符（占一个列位置）
- `2` — 宽字符（CJK 表意文字等，占两个列位置）

#### 编码方式

- 两级索引：第一级通过码点高位字节定位段，第二级通过低位字节获取宽度
- 每 2 字节编码一个宽度值（0, 1, 2）
- 覆盖 Unicode BMP (U+0000 ~ U+FFFF)

#### 使用方

- `wcwidth.c` — 查询宽字符显示宽度
- `wcswidth.c` — 查询宽字符串总宽度

#### 特殊值

- `16` (0x10) — 标记段开始/填充
- `17` (0x11) — 普通宽度 1
- `18` (0x12) — 普通宽度 1（另一种编码）
- 后续值按段编码不同宽度的字符

#### 存储结构

- 静态只读数据段（`.rodata`）
- 66 行，共约 1312 字节

#### 依赖

无。
